Knuth-Morris-Pratt 算法

Knuth-Morris-Pratt 算法是解决串的模式匹配问题的快速算法，如果给定两个字符串的长度分别为n,m。朴素字符串匹配算法的时间复杂度是O(n*m)，而KMP算法可以把时间复杂度降低到O(n+m)。

前缀，后缀，真前缀，真后缀

前缀是一个字符串的全部头部组合
后缀是一个字符串的全部尾部组合
真前缀指除了自身以外，一个字符串的全部头部组合
真后缀指除了自身以外，一个字符串的全部尾部组合

举个栗子就好理解了

str的前缀有”a”,”ab”,”abc”;
str的后缀有”c”,”bc”,”abc”;
str的真前缀有”a”,”ab”;
str的真前缀有”c”,”bc”;

朴素算法

很容易想到两个串的朴素匹配算法(时间复杂度为O(n*m))

现在假设有两个字符串S,T
S = ABBAABBBABBAABBA
T = ABBAABBA

在跑朴素算法时发现，如果两个字符串都从第一个字符开始匹配，会在最后一个字符比较时出现不相等的情况。这时朴素算法会重新从T串的第一个字符和S串的第二个字符重复地一对一匹配，直到匹配成功或者跑完S串。

S : ABBAABBBABBAABBA
T : ABBAABBA

但我们发现第一次跑到不匹配的字符时，我们就知道了S串前7个字符的与T是匹配的，所以能否利用这个信息来优化匹配的进行呢？

我们移动T串继续匹配，朴素算法是往后移动一位,那T串的第一位和S串的第二位开始匹配。但仔细观察发现，再进行下一次匹配之前我们发现如果要移动T串进行匹配至少应该保证移动一段距离后在已匹配的范围内T数组和S数组是已经匹配的。例如：
S : ABBAABBBABBAABBA
T :   ABBAABBA
这样移动在已经获知的匹配信息（加粗部分）中就已经不匹配了。所以也没必要在浪费时间去从头匹配了。

所以下一步移动应该只这样：
S : ABBAABBBABBAABBA
T :     ABBAABBA
满足已经获知的匹配信息（加粗部分）已经匹配。这样往后匹配才有可能是成功匹配的，并且可以直接从S的第8位和T的第4位开始比较了。

所以这个适当的移动应该如何计算？

next数组

S : ABBAABBBABBAABBA
T :     ABBAABBA

继续上面的思考我们发现，要满足已知信息的匹配就要保证，已匹配的字符串有有一段前缀和一段后缀是相同的，否则可以直接跳过这一段。
为了不错过任何一种情况我们应该向后移动已匹配字符串长度减去字符串中最长的相同的真前缀和真后缀的长度的距离。

因为已匹配信息必然是T串的真前缀，所以我们可以预处理一个next数组，next[i]表示长度位i的T的前缀中最长的相同的真前缀和真后缀的长度。next[i]的值是-1就代表没有相同的真前缀和真后缀。就可以在每次失配是快速移动到下一个存在匹配可能的位置。
